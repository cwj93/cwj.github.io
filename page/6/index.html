vue 数据双向绑定原理

vue 数据双向绑定主要是依赖于 Object.defineProperty( )来做数据劫持。

  • 数据双向绑定即是视图变化更新数据，数据变化更新视图。
  • 视图变化更新数据，我们可以通过事件监听来实现，比如 input 标签的’input’事件。
  • 数据变化如何更新视图 ? vue 中是通过 Object.defineProperty(obj, ‘keyName’, {})方法来实现，该方法具有 get/set 方法，var o = obj[‘keyName’]会触发 get()方法，返回处理后的值。obj[‘keyName’] = ‘赋值’，会触发 set 方法赋值。
  • 那么 vue 中就是通过 Object.defineProperty()方法来劫持 data 属性的，遍历 data 对象中的所有属性，对其属性进行劫持，然后在对应的 get()方法中，判断是否需要监听该属性，需要就将其添加到订阅器中，，在对应的 set()方法中，如果数据发生变化，那就去通知(notify())订阅者，订阅者就会执行对应的更新(update())的函数（所有订阅者都会更新）。
  • 当对数据进行更新操作时，比如 obj.key = ‘new val’ 就会触发 setter 的拦截，从而检测新值和旧值是否相等，如果相等什么也不做，如果不相等，则更新值，然后由 dep 通知 watcher 进行更新。所以，异步更新 的入口点就是 setter 中最后调用的 dep.notify() 方法。

vue3 中使用 Proxy 来实现的对象属性变化的监听

  • 通过调用 new Proxy() ，你可以创建一个代理用来替代另一个对象(被称为目标)，这个代理对目标对象进行了虚拟，因此该代理与该目标对象表面上可以被当作同一个对象来对待。
    代理允许你拦截在目标对象上的底层操作，而这原本是 JS 引擎的内部能力。拦截行为使用了一个能够响应特定操作的函数(被称为陷阱)。
  • Proxy 所带来的，是对底层操作的拦截。前面我们在实现对对象监听时使用了 Object.defineProperty，这个其实是 JS 提供给我们的高级操作，也就是通过底层封装之后暴露出来的方法。Proxy 的强大之处在于，我们可以直接拦截对代理对象的底层操作。这样我们相当于从一个对象的底层操作开始实现对它的监听
  • Object.defineProperty 监听的是对象的每一个属性，而 Proxy 监听的是对象自身
  • 使用 Object.defineProperty 需要遍历对象的每一个属性，对于性能会有一定的影响
  • Proxy 对新增的属性也能监听到，但 Object.defineProperty 无法监听到
  • 语法:
  • target:目标对象
  • handler：是一个对象，声明了代理 target 的指定行为，支持的拦截操作，一共 13 种：
    • get(target,propKey,receiver)：拦截对象属性的读取。
    • set(target,propKey,value,receiver)：拦截对象属性的设置，返回一个布尔值（修改成功）。
    • has(target,propKey)：拦截  propKey in proxy  的操作，返回一个布尔值。
    • deleteProterty(target,propKey)：拦截 delete proxy[propKey]的操作，返回一个布尔值。
    • ownKeys(target)：拦截 Object.getOwnPropertyNames(proxy)、Object.getOwnPropertySymbols(proxy)、Object.keys(proxy)、for…in  循环，返回一个数组。该方法返回目标对象所有自身的属性的属性名，而 Object.keys()的返回结果仅包括目标对象自身的可遍历属性。
    • getOwmPropertyDescript(target,propKey)：拦截 Object.getOwnPropertyDescriptor(proxy,propKey)，返回属性的描述对象。
    • defineProperty(target,propKey,propDesc)：拦截 Object.defineProperty(proxy,propKey,propDesc)、Object.defineProperties(proxy,propDesc),返回一个布尔值。
    • preventExtensions(target)：.拦截 Object.preventExtensions(proxy),返回一个布尔值。
    • getPrototypeOf(target)：拦截 Object.getPrototypeOf(proxy),返回一个对象。
    • isExtensible(target)：拦截 Object.isExtensible(proxy),返回一个布尔值。
    • setPrototypeOf(target,proto)：  拦截 Object.setPrototypeOf(proxy,proto)，返回一个布尔值。如果目标对象是函数，那么还有两种额外操作可以拦截。
    • apply(target,object,args)：拦截 Proxy 实例作为函数调用的操作，比如 proxy(…args)、proxy.call(object,…args)、proxy.apply(…)。
    • construct(target,args)：拦截 Proxy 实例作为构造函数调用的操作，比如：new proxy(…args)。

闭包

最近几天刷题都刷到与闭包有关的题目，发现对于闭包一知半解的，感觉很难受，于是打算深入了解一下闭包。

一、概念

闭包就是能够读取其他函数内部变量的函数。在js中可以理解为”函数中的函数”

面试题目

解析：a变量是对象，是引用类型。那么此时a的值指向的是对象在内存中的地址。然后a赋值给b,此时是b的值指向的也是a在内存中的地址，接着给b赋值一个基础类型，此时b的指向就变了，b的值就是基础类型，就是2

二、闭包的作用

  • 可以读取内部函数的变量
  • 变量私有化
  • 可以将变量始终保存在内存中

三、使用闭包的注意事项

  • 滥用闭包的话，可能会导致内存泄露。因为闭包会使得函数内部的变量都保存到内存中，是内存的消耗过大，会造成网页性能问题。
  • 闭包会在父函数外部，改变父函数内部变量的值

四、一道非常经典的闭包题目

解答：

  • 首先，我们仔细看看定义的fun函数，里面第一层返回了一个对象{fun: function(){}}，对象里面定义了一个fun函数，然后返回了fun(m,n)方法

  • 一、var a = fun(0); a.fun(1); a.fun(2); a.fun(3);输出什么

    • 首先，a = fun(0);此时我们根据上面定义的函数可以得知，a其实等于一个对象{fun: function(){}}。所以此时会输出undefined
    • 接着，a.fun(1);由于此时a是一个对象，里面定义了fun函数，所以此时调用的对象内部的fun方法，此方法返回了fun(m,n)函数，那么此时就相当于是调用的fun(1, 0),故输出0
    • 紧接着的都跟第二步一样，都是输出0

  • 二、var b = fun(0).fun(1).fun(2).fun(3);

    • 首先，在调用fun(0)时，得到一个对象{}，里面包含fun函数，此时fun(0)输出undefined
    • 接着，在上面的基础上调用fun(1),此时的fun函数是返回的对象的fun匿名函数，此匿名函数返回fun(m,n)函数，那么此时相当于是直接调用了fun(1,0),返回一个对象{fun: function(m){return fun(m, 1)}}，此时输出 0
    • 再接着，调用fun(2),此时调用的fun(2)返回的对象里的fun匿名函数，那么此时也就是调用的外部的fun(n,o)函数，即fun(2,1),返回的是一个对象{fun: function(m){return fun(m, 2)}},此时输出 1
    • 最后一个fun(3),调用的就是上一个的对象里的fun函数，那么此时执行的就是fun(3,2),故此时输出 2

  • 三、var c = fun(0).fun(1); c.fun(2); c.fun(3);

    • 首先，fun(0)相当于是fun(0, undefined)，返回一个对象{fun: function(m){return fun(m, 0)}},故直接输出0.
    • 接着fun(0).fun(1)就相当于是对象里的fun函数，即执行的是fun(1, 0)，返回一个对象{fun: function(m){return fun(m, 1)}},故输出 0
    • 再接着c.fun(2)就是执行的对象里的fun函数，即fun(2,1),故输出1
    • 最后c.fun(3)与c.fun(2)一样，输出 1

Object.prototype.toString.call()的使用-判断对象类型

Posted on 2021-04-06

Object.prototype.toString.call()的使用

在我们的日常开发中，时常会遇到需要做数据的类型做判断的，我们一般会使用typeof 来判断是哪一种类型的数据，但是对于对象的检测就不是那么准确了。用 typeof 是否能准确判断一个对象变量，答案是否定的，null 的结果也是 object，数组的结果也是 object。而Object.prototype.toString.call(obj)能够很好地处理这一类问题。Array 、Function等类型作为Object的实例，都重写了toString方法（Function类型返回内容为函数体的字符串，Array类型返回元素组成的字符串…..）

obj.toString()的结果和Object.prototype.toString.call(obj)的结果不一样，这是为什么？

这是因为toString为Object的原型方法，而Array 、Function等类型作为Object的实例，都重写了toString方法。不同的对象类型调用toString方法时，根据原型链的知识，调用的是对应的重写之后的toString方法（Function类型返回内容为函数体的字符串，Array类型返回元素组成的字符串…..），而不会去调用Object上原型toString方法（返回对象的具体类型），所以采用obj.toString()不能得到其对象类型，只能将obj转换为字符串类型；因此，在想要得到对象的具体类型时，应该调用Object上原型toString方法

虚拟DOM

在虚拟DOM中，一次操作需要更新10个节点的话，虚拟DOM不会立即执行DOM更行操作，而是先通过diff算法，将更新的内容保存到本地的一个js对象中，然后最终再将这个js对象一次性关联到DOM树中，再去执行浏览器的渲染流程。

用js模拟DOM节点的好处是，页面更新时，可以先全部反应在js对象（虚拟DOM）上，操作内存中的js对象的速度更快，等页面完成后，再将最终的js对象映射成真实的DOM,交给浏览器绘制

  • 1.解析DOM节点：使用js对象表示DOM节点，使用的对象的属性来表示节点的类型，属性，子节点。
  • 2.比较两个虚拟DOM树的差异：diff算法，树同层级比较，将差异存储到一个js对象中。
  • 3.将新旧虚拟DOM树的差异应用到真正的DOM树上：深层遍历DOM树，根据生成的差异js来生成真实DOM
