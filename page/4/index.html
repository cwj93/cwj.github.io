ES6 对象

es6对象中有两个新的方法，分别是Object.assign(),以及Object.is();

  • Object.assign(target, source_1, ···)

    • 用于将源对象的所有可枚举属性复制到目标对象中。
    • null 和 undefined 不能转化为对象，所以会报错
    • assign 的属性拷贝是浅拷贝
    • 数组的处理: 会将数组处理成对象，所以先将 [2,3] 转为 {0:2,1:3} ，然后再进行属性复制，所以源对象的 0 号属性覆盖了目标对象的 0。

  • Object.is(value1, value2)

    • 用来比较两个值是否严格相等，与（===）基本类似。

vue事件绑定原理

发布订阅模式

发布-订阅模式其实是一种对象间一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都将得到状态改变的通知

既然一对多 肯定有一个事件调度中心用来调度事件 订阅者可以注册事件（on）到事件中心 发布者可以发布事件（emit）到调度中心 订阅者也可以取消订阅（off）或者只订阅一次（once）

主要功能：

  • 维护一个变量 listeners，初识状态为一个空对象字面量，用于存储不同事件和事件对应的处理函数。
  • 实现$on方法：用于监听事件。
  • 实现$once方法：用于一次性事件监听。
  • 实现$emit方法：用于触发对应的事件监听。
  • 实现$off方法：用于消除对应的事件监听。

vue-router钩子函数

router 实例

router.matcher

router实例对象对外提供两个方法match(负责route匹配), addRoutes（动态添加路由）

在做路径切换transitionTo方法中，首先就会使用const route = this.router.match(location, this.current)来匹配route, 其实内部会使用matcher来做匹配。修改了matcher即新的routes生效。

对router.matcher属性做修改，即新的routes就会替换老的routes, 其实就是replaceRoutes()的含义（但是官方没有提供这个API）

router.addRoutes()

添加一条新的路由规则记录作为现有路由的子路由。如果该路由规则有 name，并且已经存在一个与之相同的名字，则会覆盖它

路由守卫分类

全局守卫

路由实例上直接操作的钩子函数，特点是所有路由配置的组件都会触发，直白点就是触发路由就会触发这些钩子函数;


路由独享守卫

直接可以在路由配置上设置的钩子函数

组件守卫

在组件内执行的钩子函数,类似于组件内的生命周期，相当于为配置路由的组件添加的生命周期钩子函数。

beforeRouteEnter（to，from， next）
在进入路由组件之前调用
在全局守卫的前置守卫和路由守卫之后触发，在全局解析守卫和全局后置守卫之前调用，访问不到组件实例，即this为undefined

beforeRouteUpdate（to，from， next）

beforeRouteLeave（to，from， next）
导航离开该组件的对应路由时调用，beforeRouteLeave可以访问组件实例this。这个离开守卫通常用来禁止用户在还未保存修改前突然离开。该导航可以通过next( false )来取消

完整的导航解析流程

  • 1.触发进入其它路由
  • 2.调用要离开路由的组件守卫beforeRouteLeave
  • 3.调用全局的前置守卫beforeEach
  • 4.在重用的组件里调用 beforeRouteUpdate
  • 5.在路由配置里调用 beforeEnter
  • 6.解析异步路由组件
  • 7.在将要进入的路由组件中调用beforeRouteEnter
  • 8.调用全局的解析守卫beforeResolve
  • 9.导航被确认
  • 10.调用全局的后置钩子afterEach。
  • 11.触发 DOM 更新mounted。
  • 12执行beforeRouteEnter守卫中传给 next的回调函数。

new的实现思路

  • 定义一个myNew函数，第一个为构造函数，其他为解构的参数
  • 首先是创建一个实例对象 let obj = {}
  • 接着生成的实例对象 继承 构造函数的原型 obj.proto = fn.prototype
  • 接着改变构造函数的this指向为实例对象 let result = fn.call(obj, …args)
  • 如果构造函数fn的执行结果（result）返回的是一个对象或者函数，那么就直接返回这个对象或函数（result）
  • 如果不是上面的条件，则直接返回实例对象obj

vue生命周期

Posted on 2021-05-27 Edited on 2021-06-08

vue生命周期

Vue 实例有一个完整的生命周期，也就是从开始创建、初始化数据、编译模版、挂载 Dom -> 渲染、更新 -> 渲染、卸载等一系列过程，我们称这是 Vue 的生命周期。

Vue 的父组件和子组件生命周期钩子函数执行顺序？

简述：即在父组件挂载，更新，销毁之前，是子组件先挂载，更新，销毁，再到父组件的挂载，更新，销毁

加载渲染过程:
父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created -> 子 beforeMount -> 子 mounted -> 父 mounted

子组件更新过程:
父 beforeUpdate -> 子 beforeUpdate -> 子 updated -> 父 updated

父组件更新过程:
父 beforeUpdate -> 父 updated

销毁过程:
父 beforeDestroy -> 子 beforeDestroy -> 子 destroyed -> 父 destroyed

父组件监听子组件的生命周期

有两种方式：

  • 一种是通过手动$emit触发父组件的事件
  • 第二种是通过@hook:mounted的形式监听子组件的生命周期

    各个生命周期的作用

  • beforeCreate: 组件实例被创建之初，组件的属性生效之前
  • created: 组件实例已经完全创建，属性也绑定，但真实 dom 还没有生成，$el 还不可用
  • beforeMount: 在挂载开始之前被调用：相关的 render 函数首次被调用
  • mounted: el 被新创建的 vm.$el 替换，并挂载到实例上去之后调用该钩子
  • beforeUpdate: 组件数据更新之前调用，发生在虚拟 DOM 打补丁之前
  • update: 组件数据更新之后
  • activited: keep-alive 专属，组件被激活时调用
  • deactivated: keep-alive 专属，组件被销毁时调用
  • beforeDestory: 组件销毁前调用 (清除watcher，子组件事件监听器)
  • destoryed: 组件销毁后调用

SPA单页应用与MPA多页应用

SPA单页应用与MPA多页应用

SPA单页应用

只有一个WEB主页面的应用，公共资源(js、css等)仅需加载一次，所有的内容都包含在主页面，对每一个功能模块组件化。单页应用跳转，就是切换相关组件，仅刷新局部资源

关键词：网站模型 、只加载单个HTML页面 、利用路由实现内容变换

描述：SPA（singgle-page application）,单页应用。是一种网站模型，是加载单个HTML页面，并在用户与应用程序交互时动态更新页面。只会在初始化的时候加载相应的HTML,CSS,JS。不会因为用户的操作而进行页面的重新加载或跳转。利用路由机制实现 HTML 内容的变换，UI 与用户的交互，避免页面的重新加载。

  • 传统页面：

    客户端 ———通过发送url请求—————— server

    客户端 ———服务器端返回html页面——— server

  • SPA页面：

    客户端 ———通过发送ajax请求—————— server

    客户端 ———服务器端返回数据——— server

优点：

  • 有良好的交互体验：提升页面交互体验，不会频繁切换浏览器页面，避免了页面的频繁重新加载
  • 前后端分离：前后端职责分离，架构清晰，前端进行交互逻辑，后端负责数据处理
  • 减轻服务器压力：服务器端只需要返回数据即可，不用管展示逻辑与页面合成
    缺点：
  • SEO（搜索引擎优化）难度较高：页面内容动态替换，页面不会刷新，导致很难被搜索引擎抓取（搜索引擎遵循http协议，#不是协议的内容）
  • 无法使用浏览器的前进后退功能，所有页面需要自己建立堆栈管理。（解决方案：利用URI中的散列+iframe实现）（配置好路由信息，通过记录浏览过的历史路由信息，可以很好的记录或历史查看过的界面）
  • 初次加载耗时多：为实现单页 Web 应用功能及显示效果，需要在加载页面的时候将 JavaScript、CSS 统一加载，部分页面按需加载

MPA多页应用

关键词：

描述：MAP（Multi-page application），多页面应用。每访问一个页面都需要重新加载HTML,JS,CSS文件，公共文件按需加载

优点：

  • 首屏加载快
  • SEO（搜索引擎优化）效果好
    缺点：
  • 页面切换慢：每一次切换页面都要重新加载html,css,js等资源

vue知识体系

1. 基础原理

1.1 Vnode

Vnode也称虚拟node节点,是对真实元素节点的抽象。“虚拟 DOM”是我们对由 Vue 组件树建立起来的整个 VNode 树的称呼


createElment函数接受三个参数，分别是：

  • 参数一：标签名、组件的选项对象、函数等等（必选）；

  • 参数二：设置这个对象的样式、属性、传的组件+的参数、绑定事件等（可选）；

  • 参数三：该节点下的其他节点，即子级虚拟节点，可以是字符串形式或数组形式，也需要使用createElement构建。

    Vnode如何检测变化并更新视图呢？

    vue是通过数据绑定来修改视图的，当某个数据发生变化时，会触发Object.defineProperty监听属性的set方法，set方法会调用Dep收集器中的通知方法notify,去通知订阅者watcher，然后订阅者warcher通过get方法调用update方法更新视图。

    更新视图diff算法：不直接对真实DOM操作。将真实DOM抽象成js对象虚拟DOM,然后操作js对象比较新旧对象的差异，经过diff算法得出一些需要修改的最小单位，再将这些小单位的视图进行更新

    diff算法是通过同层的树节点进行比较而非对树进行逐层搜索遍历的方式

1.2 nextTick

任务：
分为同步任务和异步任务

事件循环：
就是从script整体代码（宏任务）开始，遇到同步代码就进入主线程中执行，遇到异步代码则分发任务，setTimeout等会进入下一事件循环宏任务队列中，Promise.then()则会进入当前的事件循环的微任务队列，当前事件循环宏任务执行完毕，接着执行当前事件循环的微任务队列，然后微任务队列执行完毕之后，进入下一事件循环执行宏任务队列。

事件循环的执行顺序为：

宏任务 -> 微任务队列 -> 下一事件循环的宏任务队列(setTimeout) -> 下一事件循环的微任务队列

nextTick的目的是将回调函数的调用延迟到下一次DOM更新数据之后。在vue中，修改数据之后并不会立即更新DOM,因为DOM的更新是异步的，无法通过同步代码获取，需要使用nextTick，在下一次事件循环中获取（Vue中Dom的更新是异步的）

1.3 MVVM

MVC：

  • Model + View + Controller
  • Model: 数据层(模型)，负责数据的处理和获取的数据接口层(是应用程序中用于处理应用程序数据逻辑的部分。通常模型对象负责在数据库中存取数据)
  • View: 视图层，是应用程序中处理数据显示的部分,通常视图是依据模型数据创建的
  • Controller: 控制器层(应用程序中处理用户交互的部分)，通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据
  • MVC 的思想：一句话描述就是 Controller 负责将 Model 的数据用 View 显示出来，换句话说就是在 Controller 里面把 Model 的数据赋值给 View。

MVVM:

  • Model + View + ViewModel
  • Model: 数据层(模型)，负责数据的处理和获取的数据接口层（Model 是指数据模型，泛指后端进行的各种业务逻辑处理和数据操控，对于前端来说就是后端提供的 api 接口）
  • View: 视图层，负责给用户提供操作界面
  • ViewModel: 数据视图层(实现了数据的双向绑定，实现的方式是：DOM 事件监听)，负责处理View层的业务逻辑，做好绑定属性的监听，当ViewModel数据发生变化，View层就会得到更新。
  • 优点：
    • 实现了 View 和 Model 的自动同步
    • 分离View层与Model，降低代码耦合，提高视图或者逻辑的重用性
    • 自动更新DOM:利用双向数据绑定，数据更新后视图更新，减少手动操作DOM次数
    • 只需要关注数据的变化，无需关注DOM操作
  • 缺点：
    • 可能比较难调试。vue数据绑定的声明是指令式写在vue模板中的，无法debugger调试
    • 严格的 MVVM 要求 View 不能和 Model 直接通信，而 Vue 提供了$refs 这个属性，让 Model 可以直接操作 View，违反了这一规定，所以说 Vue 没有完全遵循 MVVM
