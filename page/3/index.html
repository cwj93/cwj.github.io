uniapp的使用

uniapp是一个跨平台的框架。可以用于开发小程序、H5、原生app等等。

上手：

  • 通过使用HbuilderX(编辑器)可视化页面创建项目
  • 直接在编辑器中新建项目，然后选择uni-ui项目内置项目模板
  • 运行项目的话，则是直接通过编辑器的运行功能运行对应的设备即可
  • 在编辑器中选择发行，即可打包发布

注意：

  • pages.json 文件用来对 uni-app 进行全局配置，决定页面文件的路径、窗口样式、原生的导航栏、底部的原生tabbar ;

  • pages.json类似微信小程序中app.json的页面管理部分。注意定位权限申请等原属于app.json的内容，在uni-app中是在manifest中配置

  • manifest.json 文件是应用的配置文件，用于指定应用的名称、图标、权限等。HBuilderX 创建的工程此文件在根目录，CLI 创建的工程此文件在 src 目录

生命周期：

  • 应用生命周期
  • 页面生命周期
  • 组件生命周期(vue的生命周期)

H5与APP的区别

Posted on 2021-06-08

H5与APP的区别

1.开发不同

app开发是基于移动操作系统的，比如安卓，ios；而H5是在运行在浏览器的应用，不受移动操作系统的限制

2.运行能力有差距

移动app可以完全发挥手机的硬件优势，比如摄像头，本地信息等；而H5只能使用少量的硬件功能

3.获取来源不同

移动app是需要下载安装到本地才能使用；而H5只需要手机可以访问浏览器即可使用

4.存储容量不同

移动app是安装到手机本地的，使用的手机的内存容量；而H5是在浏览器运行的，消耗的是服务器的内存，缺点是对网络的要求较高

vue中hash模式与history模式的区别

vue路由的两种模式核心都是-改变视图的同时不向服务发送请求

hash 模式

hash模式的的url中会带有#号，#号虽然出现在url中，但是在发送http请求时，不会被包括在http请求中，对后端没有任何影响。因此改变hash不会重新加载页面。
会使用到的api：

  • hash改变会触发hashchange事件

存在的问题：

  • hash的传参是基于url的，如果要传递复杂的数据，会有体积的限制
  • hash 本来是拿来做页面定位的，如果拿来做路由的话，原来的锚点功能就不能用了

    history 模式

    history模式的URL中不带有#号，该模式是使用pushState,replaceState来修改URL的，会触发popstate事件，我们只需要对popstate事件进行监听获取location.pathname,根据对应的参数修改页面的内容即可。

会使用到的api：

  • pushState,replaceState

存在的问题：

  • history模式需要后端配合。因为在刷新页面之后，浏览器会发送HTTP请求，如果后端没有匹配该Url的话，就会返回404.

总结

1 hash 模式下，仅 hash 符号之前的内容会被包含在请求中，如 http://www.abc.com，因此对于后端来说，即使没有做到对路由的全覆盖，也不会返回 404 错误。

2 history 模式下，前端的 URL 必须和实际向后端发起请求的 URL 一致，如 http://www.abc.com/book/id。如果后端缺少对 /book/id 的路由处理，将返回 404 错误。Vue-Router 官网里如此描述：“不过这种模式要玩好，还需要后台配置支持……所以呢，你要在服务端增加一个覆盖所有情况的候选资源：如果 URL 匹配不到任何静态资源，则应该返回同一个 index.html 页面，这个页面就是你 app 依赖的页面。”

3 结合自身例子，对于一般的 Vue + Vue-Router + Webpack + XXX 形式的 Web 开发场景，用 history 模式即可，只需在后端（Apache 或 Nginx）进行简单的路由配置，同时搭配前端路由的 404 页面支持。

ES6中的数组以及对象的方法

ES6 对象

es6对象中有两个新的方法，分别是Object.assign(),以及Object.is();

  • Object.assign(target, source_1, ···)

    • 用于将源对象的所有可枚举属性复制到目标对象中。
    • null 和 undefined 不能转化为对象，所以会报错
    • assign 的属性拷贝是浅拷贝
    • 数组的处理: 会将数组处理成对象，所以先将 [2,3] 转为 {0:2,1:3} ，然后再进行属性复制，所以源对象的 0 号属性覆盖了目标对象的 0。

  • Object.is(value1, value2)

    • 用来比较两个值是否严格相等，与（===）基本类似。
    • 基本用法 //一是+0不等于-0
      Object.is(+0,-0); //false
      +0 === -0 //true
      //二是NaN等于本身
      Object.is(NaN,NaN); //true
      NaN === NaN //false console.log(Array.of(1, 2, 3, 4)); // [1, 2, 3, 4]

vue事件绑定原理

发布订阅模式

发布-订阅模式其实是一种对象间一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都将得到状态改变的通知

既然一对多 肯定有一个事件调度中心用来调度事件 订阅者可以注册事件（on）到事件中心 发布者可以发布事件（emit）到调度中心 订阅者也可以取消订阅（off）或者只订阅一次（once）

主要功能：

  • 维护一个变量 listeners，初识状态为一个空对象字面量，用于存储不同事件和事件对应的处理函数。
  • 实现$on方法：用于监听事件。
  • 实现$once方法：用于一次性事件监听。
  • 实现$emit方法：用于触发对应的事件监听。
  • 实现$off方法：用于消除对应的事件监听。

vue-router钩子函数

Posted on 2021-06-01 Edited on 2021-07-07

vue-router钩子函数

router 实例

router.matcher

router实例对象对外提供两个方法match(负责route匹配), addRoutes（动态添加路由）

在做路径切换transitionTo方法中，首先就会使用const route = this.router.match(location, this.current)来匹配route, 其实内部会使用matcher来做匹配。修改了matcher即新的routes生效。

对router.matcher属性做修改，即新的routes就会替换老的routes, 其实就是replaceRoutes()的含义（但是官方没有提供这个API）

router.addRoutes()

添加一条新的路由规则记录作为现有路由的子路由。如果该路由规则有 name，并且已经存在一个与之相同的名字，则会覆盖它

路由守卫分类

全局守卫

路由实例上直接操作的钩子函数，特点是所有路由配置的组件都会触发，直白点就是触发路由就会触发这些钩子函数;


路由独享守卫

直接可以在路由配置上设置的钩子函数

组件守卫

在组件内执行的钩子函数,类似于组件内的生命周期，相当于为配置路由的组件添加的生命周期钩子函数。

beforeRouteEnter（to，from， next）
在进入路由组件之前调用
在全局守卫的前置守卫和路由守卫之后触发，在全局解析守卫和全局后置守卫之前调用，访问不到组件实例，即this为undefined

beforeRouteUpdate（to，from， next）

beforeRouteLeave（to，from， next）
导航离开该组件的对应路由时调用，beforeRouteLeave可以访问组件实例this。这个离开守卫通常用来禁止用户在还未保存修改前突然离开。该导航可以通过next( false )来取消

完整的导航解析流程

  • 1.触发进入其它路由
  • 2.调用要离开路由的组件守卫beforeRouteLeave
  • 3.调用全局的前置守卫beforeEach
  • 4.在重用的组件里调用 beforeRouteUpdate
  • 5.在路由配置里调用 beforeEnter
  • 6.解析异步路由组件
  • 7.在将要进入的路由组件中调用beforeRouteEnter
  • 8.调用全局的解析守卫beforeResolve
  • 9.导航被确认
  • 10.调用全局的后置钩子afterEach。
  • 11.触发 DOM 更新mounted。
  • 12执行beforeRouteEnter守卫中传给 next的回调函数。

new的实现

new的实现思路

  • 定义一个myNew函数，第一个为构造函数，其他为解构的参数
  • 首先是创建一个实例对象 let obj = {}
  • 接着生成的实例对象 继承 构造函数的原型 obj.proto = fn.prototype
  • 接着改变构造函数的this指向为实例对象 let result = fn.call(obj, …args)
  • 如果构造函数fn的执行结果（result）返回的是一个对象或者函数，那么就直接返回这个对象或函数（result）
  • 如果不是上面的条件，则直接返回实例对象obj
