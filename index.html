this的指向

this的指向在函数创建的时候是决定不了的，在调用的时候才能决定，谁调用的就指向谁（this永远指向的是最后调用它的对象）

  • 1.全局作用域下，this执行window
  • 2.函数作用域下，在非严格模式下，this指向window
  • 3.函数作用域下，在严格模式下，this执行undefined
  • 4.在对象里面，this指向的是当前对象
  • 5.箭头函数的this指向在创建时就已确定（箭头函数不能作为构造函数）
  • 6.new 关键字调用函数，指向是创建的新对象
  • 7.在定时器函数中，this指向window
  • 8.通过call、apply、bind修改this指向，指向绑定的对象

BFC 块级格式化上下文

  • 概念： css盒模型的渲染规则
  • 渲染规则：（html标签就是一个BFC元素）
    • 规则1：创建的BFC元素，在垂直方向上的margin会发生重叠（塌陷）
    • 规则2：BFC元素在页面上独立的容器，外面的元素和里面的元素互不影响
    • 规则3：BFC元素不会和浮动的元素重叠（可以解释两栏自适应）
    • 规则4：计算BFC元素的高度时，里面浮动的元素的高度也会参与计算（用来解释overflow：hidden可以清楚浮动）
  • 普通元素创建BFC
    • float浮动属性不为none
    • overflow: hidden / auto
    • position: 不为static 、 relative都可以
    • display: table-cell … 表格相关的

css3新特性

1.过渡 transition

  • 作用： 给元素添加一个过渡效果，让页面效果不过于生硬
  • 语法：transition： CSS属性，花费时间，效果曲线(默认ease)，延迟时间(默认0)
  • 例子：transition：width,.5s,ease,.2s / transition：all,.5s
  • 应用场景：下拉的显示/隐藏的过渡 元素hover时一些过渡的效果，如颜色等

2.动画 animation

  • 作用：做一个预设的动画和一些页面交互的动画效果。给元素添加一个动画效果，让页面效果不过于生硬
  • 语法：animation：动画名称，一个周期花费时间，运动曲线（默认ease），动画延迟（默认0），播放次数（默认1），是否反向播放动画（默认normal），是否暂停动画（默认running）
  • 例子： animation: logo2-line 2s linear; //执行一次logo2-line动画，运动时间2秒，运动曲线为 linear
  • 重要属性：animation-fill-mode 属性规定动画在播放之前或之后，其动画效果是否可见
  • 用法： animation-fill-mode : none | forwards | backwards | both;

vue v-for中key的作用

高效地更新虚拟DOM

v-for中的key是为循环里面的每一个虚拟DOM节点添加一个唯一的标识值，是为了跟踪每一个节点，便于在DOM更新的时候能够高效地更新虚拟DOM。

vue中v-for更新已渲染过的元素列表时，采用的是【就地复用】的策略。当数据项顺序发生变化时，vue中不是移动DOM元素来匹配数据项的改变，而是简单复用此处的每一个元素，并且确保它在特定索引下显示已渲染过的每一个元素。

key 的特殊属性主要用在Vue的虚拟DOM算法，在新旧nodes对比时辨识VNodes。如果不使用 key，Vue会使用一种【最大限度减少动态元素】并且【尽可能的尝试修复/再利用相同类型元素】的算法。使用key，它会基于key的变化重新排列元素顺序，并且会移除 key 不存在的元素。

vue-$nextTick的作用

Posted on 2021-06-15

$nextTick的作用

$nextTick是在【下次DOM更新循环】结束之后执行的【延迟回调】，在修改数据之后使用$nextTick,则可以在回调中获取更新后的DOM.

vue中实现响应式并不是数据发生变化之后DOM就立即变化的，而是按照一定的策略（同一事件循环的所有数据变化完成后，再异步进行视图更新）来进行DOM更新的

【$nextTick的实现】中使用到了【ES6的Promise】及【HTML5的MutationObserver】，当环境不支持时，使用【setTimeout(fn, 0)】替代。上述的三种方法，均为【异步API】。其中MutationObserver类似事件，又有所区别；事件是同步触发，其为异步触发，即DOM发生变化之后，不会立刻触发，等当前所有的DOM操作都结束后触发

vue-图片懒加载的实现原理

图片懒加载的优点

  • 提升页面渲染效率，避免大量图片加载造成页面卡顿。
  • 按需加载，减少无效的图片加载，节约网络资源。

图片懒加载插件 vue-lazyload

自定义指令的方式实现图片懒加载

首先，我们会先定义一个变量(let imgs = [])存储所有的图片数据，然后准备一张占位图，为每一张图片都默认使用的是占位图；接着监听图片滚动事件（scroll）去触发(eventBus.$emit())事件总线中的注册的”isScroll”事件；为了避免滚动过程中加载不必要的图片，所以需要做防抖处理，待滚动停止后的某个时间点再执行图片加载函数（根据图片是否在图片容器可视区内）。然后加载过了的图片需要从imgs中清除，避免二次加载

uniapp的使用

Posted on 2021-06-08

uniapp的使用

uniapp是一个跨平台的框架。可以用于开发小程序、H5、原生app等等。

上手：

  • 通过使用HbuilderX(编辑器)可视化页面创建项目
  • 直接在编辑器中新建项目，然后选择uni-ui项目内置项目模板
  • 运行项目的话，则是直接通过编辑器的运行功能运行对应的设备即可
  • 在编辑器中选择发行，即可打包发布

注意：

  • pages.json 文件用来对 uni-app 进行全局配置，决定页面文件的路径、窗口样式、原生的导航栏、底部的原生tabbar ;

  • pages.json类似微信小程序中app.json的页面管理部分。注意定位权限申请等原属于app.json的内容，在uni-app中是在manifest中配置

  • manifest.json 文件是应用的配置文件，用于指定应用的名称、图标、权限等。HBuilderX 创建的工程此文件在根目录，CLI 创建的工程此文件在 src 目录

生命周期：

  • 应用生命周期
  • 页面生命周期
  • 组件生命周期(vue的生命周期)

H5与APP的区别

Posted on 2021-06-08

H5与APP的区别

1.开发不同

app开发是基于移动操作系统的，比如安卓，ios；而H5是在运行在浏览器的应用，不受移动操作系统的限制

2.运行能力有差距

移动app可以完全发挥手机的硬件优势，比如摄像头，本地信息等；而H5只能使用少量的硬件功能

3.获取来源不同

移动app是需要下载安装到本地才能使用；而H5只需要手机可以访问浏览器即可使用

4.存储容量不同

移动app是安装到手机本地的，使用的手机的内存容量；而H5是在浏览器运行的，消耗的是服务器的内存，缺点是对网络的要求较高
