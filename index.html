vue数据双向绑定原理

vue数据双向绑定主要是依赖于Object.defineProperty( )来做数据劫持。

  • 数据双向绑定即是视图变化更新数据，数据变化更新视图。
  • 视图变化更新数据，我们可以通过事件监听来实现，比如input标签的’input’事件。
  • 数据变化如何更新视图 ? vue中是通过Object.defineProperty(obj, ‘keyName’, {})方法来实现，该方法具有get/set方法，var o = obj[‘keyName’]会触发get()方法，返回处理后的值。obj[‘keyName’] = ‘赋值’，会触发set方法赋值。
  • 那么vue中就是通过Object.defineProperty()方法来劫持data属性的，遍历data对象中的所有属性，对其属性进行劫持，然后在对应的get()方法中，判断是否需要监听该属性，需要就将其添加到订阅器中，，在对应的set()方法中，如果数据发生变化，那就去通过订阅者，订阅者就会执行对应的更新的函数（所有订阅者都会更新）。

Object.defineProperty( )

闭包

最近几天刷题都刷到与闭包有关的题目，发现对于闭包一知半解的，感觉很难受，于是打算深入了解一下闭包。

一、概念

闭包就是能够读取其他函数内部变量的函数。在js中可以理解为”函数中的函数”

二、闭包的作用

  • 可以读取内部函数的变量
  • 可以将变量始终保存在内存中

三、使用闭包的注意事项

  • 滥用闭包的话，可能会导致内存泄露。因为闭包会使得函数内部的变量都保存到内存中，是内存的消耗过大，会造成网页性能问题。
  • 闭包会在父函数外部，改变父函数内部变量的值

四、一道非常经典的闭包题目

解答：

  • 首先，我们仔细看看定义的fun函数，里面第一层返回了一个对象{fun: function(){}}，对象里面定义了一个fun函数，然后返回了fun(m,n)方法

  • 一、var a = fun(0); a.fun(1); a.fun(2); a.fun(3);输出什么

    • 首先，a = fun(0);此时我们根据上面定义的函数可以得知，a其实等于一个对象{fun: function(){}}。所以此时会输出undefined
    • 接着，a.fun(1);由于此时a是一个对象，里面定义了fun函数，所以此时调用的对象内部的fun方法，此方法返回了fun(m,n)函数，那么此时就相当于是调用的fun(1, 0),故输出0
    • 紧接着的都跟第二步一样，都是输出0

  • 二、var b = fun(0).fun(1).fun(2).fun(3);

    • 首先，在调用fun(0)时，得到一个对象{}，里面包含fun函数，此时fun(0)输出undefined
    • 接着，在上面的基础上调用fun(1),此时的fun函数是返回的对象的fun匿名函数，此匿名函数返回fun(m,n)函数，那么此时相当于是直接调用了fun(1,0),返回一个对象{fun: function(m){return fun(m, 1)}}，此时输出 0
    • 再接着，调用fun(2),此时调用的fun(2)返回的对象里的fun匿名函数，那么此时也就是调用的外部的fun(n,o)函数，即fun(2,1),返回的是一个对象{fun: function(m){return fun(m, 2)}},此时输出 1
    • 最后一个fun(3),调用的就是上一个的对象里的fun函数，那么此时执行的就是fun(3,2),故此时输出 2

  • 三、var c = fun(0).fun(1); c.fun(2); c.fun(3);

    • 首先，fun(0)相当于是fun(0, undefined)，返回一个对象{fun: function(m){return fun(m, 0)}},故直接输出0.
    • 接着fun(0).fun(1)就相当于是对象里的fun函数，即执行的是fun(1, 0)，返回一个对象{fun: function(m){return fun(m, 1)}},故输出 0
    • 再接着c.fun(2)就是执行的对象里的fun函数，即fun(2,1),故输出1
    • 最后c.fun(3)与c.fun(2)一样，输出 1

第161题 用最精炼的代码实现数组非零非负最小值 index

实现：

  • 首先，仔细阅读题目要求，要求的是非零非负，那么就可以用大于0(> 0)表示
  • 其次，要求是最小值，那么我们就应该想到直接使用Math.min()方法,要注意min()方法的参数不是数组哈，需要将数组解构
  • 接着，要求输出的是最小值的索引值index，那么数组有一个方法是可以返回数组中元素所在的索引findIndex().

此处可以扩展一下find()和findIndex()方法的比较

  • find()–find()函数用来查找目标元素，找到就返回该元素，找不到返回undefined
  • findIndex()–findIndex()函数也是查找目标元素，找到就返回元素的位置，找不到就返回-1
  • 都是查找回调函数,都含有三个参数，
    • value–每一次迭代查找的数组元素
    • index–每一次迭代查找的数组元素索引
    • arr–被查找的数组
  • 语法
    • [1,2,3,4].find((value, index, arr) => { return value > 1}) // 2
    • [1,2,3,4].findIndex((value, index, arr) => { return value > 1}) // 1

Object.prototype.toString.call()的使用-判断对象类型

Posted on 2021-04-06

Object.prototype.toString.call()的使用

在我们的日常开发中，时常会遇到需要做数据的类型做判断的，我们一般会使用typeof 来判断是哪一种类型的数据，但是对于对象的检测就不是那么准确了。用 typeof 是否能准确判断一个对象变量，答案是否定的，null 的结果也是 object，数组的结果也是 object。而Object.prototype.toString.call(obj)能够很好地处理这一类问题。Array 、Function等类型作为Object的实例，都重写了toString方法（Function类型返回内容为函数体的字符串，Array类型返回元素组成的字符串…..）

obj.toString()的结果和Object.prototype.toString.call(obj)的结果不一样，这是为什么？

这是因为toString为Object的原型方法，而Array 、Function等类型作为Object的实例，都重写了toString方法。不同的对象类型调用toString方法时，根据原型链的知识，调用的是对应的重写之后的toString方法（Function类型返回内容为函数体的字符串，Array类型返回元素组成的字符串…..），而不会去调用Object上原型toString方法（返回对象的具体类型），所以采用obj.toString()不能得到其对象类型，只能将obj转换为字符串类型；因此，在想要得到对象的具体类型时，应该调用Object上原型toString方法
