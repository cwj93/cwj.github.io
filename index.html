vuex 的实现原理

vuex 是 vue 的一个插件，vuex 只能用于 vue 上，因为其高度依赖于 vue 的双向绑定和插件系统

  • vuex 的注入代码是调用了 applyMixin 方法，其实就是调用了 vue.mixin，在所有组件的 beforeCreate 生命周期注入了设置 this.$store 这样的一个对象
  • vuex.Store 的实现就是通过调用 new Vue 实例对象，通过 vue 的双向数据绑定然后注入 vue 的 data 属性中的$$state 属性中

总结

  • Vuex 的双向绑定通过调用 new Vue 实现，然后通过 Vue.mixin 注入到 Vue 组件的生命周期中，再通过劫持 state.get 将数据放入组件中

  • 本质上 vuex 内部其实是 new Vue 实现，vuex 中的 state 放到 vue 的 data 中的$$state 中，getters 是通过 new Vue 的 computed 实现，getters 的取值本质上是将所有的方法放到 computed 中，getters 取值的时候取的就是 computed

再次总结

vuex是通过调用applyMixin方法注入到vue中，在applyMixin方法中实际上是使用了vue.mixin在所有组件的beforeCreate生命周期中注入到this.$store这样的一个对象中，然后在vue中通过this调用store,实际上是获取的this._vm._data.$$state

什么是webpack ？

  • 它是一个前端资源加载/打包工具。
  • 将根据模块得依赖关系进行静态分析，并依据规则生成对应的静态资源。
  • 分析你的项目结构，找到javascript模块以及其他的一些浏览器不能直接运行的拓展语言（Scss，TypeScript等），并将其打包为合适的格式以供浏览器使用。
  • 从一个默认入口文件（js、json），找到所有依赖关系的模块，然后把它们转换成浏览器可以使用的文件。

安装?

  • 在安装webpack前，本地环境需要已安装了node.js，然后按照下列步骤进行安装：
  • 检查npm和node版本，确定全局是否安装。
    • npm -v
    • node -v
  • 全局安装webpack(不推荐，安装后版本就固定了，webpack版本更新比较快，推荐局部安装，项目之间不影响)
    • npm install webpack -g
    • 查看webpack版本：webpack -v
    • 如果提示需要安装webpack-cli，输入下面命令：
    • npm install webpack-cli -g
  • 局部安装
    • 同全局安装方式，把-g换成-s
    • 查看webpack版本：./node_modules/.bin/webpack -v
    • 查看webpack版本：npx webpack -v
  • 查看webpack历史版本信息
    • npm info webpack

webpack配置文件 ？

  • 默认的webpack配置文件 webpack.config.js
  • 不使用默认的配置文件
    • 启动 npx webpack –config 配置文件文件名称.js
  • 修改package.json script字段，可通过npm run来启动，原理就是模块局部安装会在node_modules/.bin目录下创建一个软链接
  • 基础配置
  • 问题1:development和production的区别
    • 开发环境打包后的代码没有压缩，生产环境打包后的代码是压缩过的。
    • 开发环境会将DefinePlugin中process.env.NODE_ENV的值设置为development，生产环境会将DefinePlugin中process.env.NODE_ENV的值设置为production
    • 启动不同的默认插件

loader ?

  • 如何配置

  • file-loader :处理静态资源（图片、字体）

  • url-loader:强化版的file-loader，可以设置limit参数，小于limit的值则以base64的形式打包到js代码中

  • style-loader 在html的head标签里创建一个style标签，然后将css-loader合并的样式放进去
  • css-loader 将css文件合并
  • less-loader css预处理器
  • postcss-loader 自动添加前缀

文件监听 ？

  • 启动webpack命令
  • 在配置文件设置

执行npm run xxx的时候发生了什么

Posted on 2022-05-25

执行npm run xxx的时候发生了什么

过程

  • 一：执行 npm run serve
  • 二：找到项目中的package.json文件
  • 三：找到package.json文件中的scripts配置
  • 四：找到scripts配置中serve对应的命令
    • serve: ‘vue-cli-service serve’
    • 问题：为什么要执行npm run serve，而不是直接执行vue-cli-service serve呢？
      • 因为vue-cli-service在操作系统中找不到vue-cli-service这个指令，直接执行vue-cli-service 会报错
    • 问题：既然vue-cli-service这条指令不存在操作系统中，为什么执行npm run serve的时候，也就是相当于执行了vue-cli-service serve ，为什么这样它就能成功，而且不报指令不存在的错误呢？
      • 因为npm i 的时候，npm 就帮我们把这种软连接配置好了，其实这种软连接相当于一种映射，执行npm run xxx 的时候，就会到 node_modules/bin中找对应的映射文件，然后再找到相应的js文件来执行。
  • 五：通过scripts配置中serve对应的命令去./node_modules/.bin找到对应的vue-cli-service文件作为脚本执行,相当于执行了 ./node_modules/.bin/vue-cli-service serve（最后的 serve 作为参数传入）

总结

+ 1、运行 npm run xxx的时候，npm 会先在当前目录的 node_modules/.bin 查找要执行的程序，如果找到则运行；
+ 2、没有找到则从全局的 node_modules/.bin 中查找，npm i -g xxx就是安装到到全局目录；
+ 3、如果全局目录还是没找到，那么就从 path 环境变量中查找有没有其他同名的可执行程序。

vue-使用Vue.observable()进行状态管理

用法：

让一个对象可响应。Vue 内部会用它来处理 data 函数返回的对象。

返回的对象可以直接用于渲染函数和计算属性内，并且会在发生变更时触发相应的更新。也可以作为最小化的跨组件状态存储器，用于简单的场景

注意事项：

在 Vue 2.x 中，被传入的对象会直接被 Vue.observable 变更，所以如这里展示的，它和被返回的对象是同一个对象。在 Vue 3.x 中，则会返回一个可响应的代理，而对源对象直接进行变更仍然是不可响应的。因此，为了向前兼容，我们推荐始终操作使用 Vue.observable 返回的对象，而不是传入源对象。
