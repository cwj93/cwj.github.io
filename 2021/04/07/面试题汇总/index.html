第161题 用最精炼的代码实现数组非零非负最小值 index

实现：

  • 首先，仔细阅读题目要求，要求的是非零非负，那么就可以用大于0(> 0)表示
  • 其次，要求是最小值，那么我们就应该想到直接使用Math.min()方法,要注意min()方法的参数不是数组哈，需要将数组解构
  • 接着，要求输出的是最小值的索引值index，那么数组有一个方法是可以返回数组中元素所在的索引findIndex().

此处可以扩展一下find()和findIndex()方法的比较

  • find()–find()函数用来查找目标元素，找到就返回该元素，找不到返回undefined
  • findIndex()–findIndex()函数也是查找目标元素，找到就返回元素的位置，找不到就返回-1
  • 都是查找回调函数,都含有三个参数，
    • value–每一次迭代查找的数组元素
    • index–每一次迭代查找的数组元素索引
    • arr–被查找的数组
  • 语法
    • [1,2,3,4].find((value, index, arr) => { return value > 1}) // 2
    • [1,2,3,4].findIndex((value, index, arr) => { return value > 1}) // 1

第 160 题：输出以下代码运行结果，为什么？如果希望每隔 1s 输出一个结果，应该如何改造？注意不可改动 square 方法

实现：forEach是不能阻塞的，默认是请求并行发起，所以是同时输出1、4、9

  • forEach是不能阻塞的，默认是请求并行发起

扩展：

  • 数组的其他常用循环方法
    • forEach()–不能终止循环(无法使用break语句或者return语句终止)，forEach方法对数组的每个元素执行一次提供的函数，forEach循环里不能有break或continue， 会产生报错 callback可以使用return但return的结果没有作用，只是提前结束函数运行，也就是calback总是返回undefined forEach返回值undefined
    • for of–for…of语句在可迭代对象上创建一个迭代循环，执行语句; 和for循环一样，会等待await执行完后继续执行
    • forEach和map,await不生效；使用break或continue会报错
    • for循环、for…in,for…of，支持await，for和for…of中可以使用break和continue；for…in会忽略continue和break for of 语法 axios 源码里所用到的，利用 promise 本身的链式调用来实现串行

第 158 题：如何模拟实现 Array.prototype.splice

实现：

  • 首先，要了解原生splice的定义,splice() 方法向/从数组中添加/删除项目，然后返回被删除的项目。
  • 接着，了解其语法，arrayObject.splice(index,howmany,item1,…..,itemX)
    • index - 必需。整数，规定添加/删除项目的位置，使用负数可从数组结尾处规定位置。
    • howmany - 必需。要删除的项目数量。如果设置为 0，则不会删除项目。
    • item1, …, itemX - 可选。向数组添加的新项目。

第 156 题：求最终 left、right 的宽度（变形）

实现：

  • 先了解该题目要考察的是什么。该题目考察的flex布局的知识点，考察对css的flex属性的了解
  • flex: flex-grow flex-shrink flex-basis|auto|initial|inherit;
    • flex-grow: 一个数字，规定项目将相对于其他灵活的项目进行扩展的量
    • flex-shrink: 一个数字，规定项目将相对于其他灵活的项目进行收缩的量
    • flex-basis: 项目的长度。合法值：”auto”、”inherit” 或一个后跟 “%”、”px”、”em” 或任何其他长度单位的数字
    • auto: 与 1 1 auto 相同。
    • none: 与 0 0 auto 相同。
    • initial: 设置该属性为它的默认值，即为 0 1 auto。请参阅 initial。
    • inherit: 从父元素继承该属性。请参阅 inherit。
  • 在flex-basis设置了具体的长度之后，当flex-basis的总和未撑满容器时，执行的是flex-grow，反之执行flex-shrink
